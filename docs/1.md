# Psychology Buddy - AI-Powered Mental Health Platform

## Project Overview

Psychology Buddy is a comprehensive mental health and wellness platform built for educational institutions. It provides students with AI-driven psychological support, educational content, self-help tools, and a monitored, school-controlled environment for mental health management.

**Current Version**: 0.1.0  
**Framework**: Next.js 16 + React 19 + TypeScript  
**Database**: PostgreSQL with Prisma ORM  
**Styling**: Tailwind CSS + Radix UI  
**Authentication**: NextAuth.js with custom session management

## RBAC (Role-Based Access Control) System

### RBAC Implementation Files

| File | Purpose |
|------|---------|
| `src/config/permission.ts` | Central permission definitions, role-to-permission mapping |
| `src/middleware/permission.middleware.ts` | API route protection with `withPermission()` wrapper |
| `src/hooks/usePermissions.ts` | Frontend permission checking hook |
| `src/hooks/useUserPermissions.ts` | Alternative frontend permission hook |
| `src/components/Protected.tsx` | Role-based UI rendering component |
| `src/lib/permissions.ts` | Permission utility functions |

### RBAC Middleware Usage

```typescript
// Protect API routes
export const POST = withPermission({ 
  module: 'PSYCHO_EDUCATION', 
  action: 'CREATE' 
})(createArticleHandler);

export const DELETE = withPermission({ 
  module: 'USER_MANAGEMENT', 
  action: 'DELETE' 
})(deleteUserHandler);
```

### Frontend Permission Usage

```tsx
// Permission-based rendering
if (hasPermission('users.create')) {
  // Show create user button
}

// Role-based rendering
<SuperAdminOnly>
  <SystemSettings />
</SuperAdminOnly>
```

### Security Rules

**Students Can**: View educational content, use self-help tools, track mood, earn badges.  
**Students Cannot**: Create/delete content, manage users, access analytics, view escalations, change settings.

**Admins Can**: Manage school users (no deletion), create content, view analytics, respond to escalations.  
**Admins Cannot**: Delete users, manage access control, access other schools' data, monitor chats.

**SuperAdmin**: Bypasses all permission checks. Full system access.

## Current System Features

### 1. AI-Powered Chat Support
- **Student Chat**: AI-driven conversational support with session persistence
- **Session Isolation**: Student-specific sessionStorage keys prevent cross-student data leakage
- **Automatic Termination**: Time-based and conversation-based session ending
- **Emergency Detection**: AI identifies concerning content and triggers escalations

### 2. Psychoeducation Library
- Block-based article editor (7 content types)
- Two-stage creation: Metadata -> Content
- Draft / Preview / Published workflow
- Mobile-responsive student reading view
- Category and label management

### 3. Self-Help Tools Suite
- **Journaling**: Writing, audio, and art journals with mood tracking
- **Music Therapy**: Curated therapeutic playlists with mood-based recommendations
- **Meditation**: Guided sessions, breathing exercises, mindfulness practices
- **Progress Analytics**: Visual trends and insights

### 4. Gamification & Engagement
- **Badge System**: 6 badge types (Streak, Journal Count, Article Read, Meditation, Music, Mood Check-in)
- **Streak Tracking**: Daily activity monitoring with consecutive day calculation
- **Automatic Awarding**: AI-driven badge allocation based on activity

### 5. Escalation & Safety System
- **AI Detection**: Automated identification of concerning content
- **Multi-Level Alerts**: Low, Medium, High, Critical severity levels
- **Email Notifications**: Automatic email alerts to admins (Gmail SMTP configured)
- **Daily Summary Emails**: Automated 9:00 AM daily reports to all admins
- **Real-Time Alerts**: Immediate in-app notifications for critical situations

### 6. User Management
- **Student Profiles**: CRUD with mood tracking and journaling history
- **Admin Management**: Role assignment, permission management
- **Counselor Management**: School-mandatory assignment with UUID validation
- **Account Status**: ACTIVE, INACTIVE, SUSPENDED states enforced at middleware level

### 7. School & Organization Management
- **Multi-Tenant**: Strict school data isolation
- **Location System**: Multi-location support within schools
- **Primary Admin Assignment**: Per-school primary administrator designation
- **School Scoping**: Admins can only access their assigned school's data

### 8. Analytics & Reporting
- **Platform Metrics**: DAU, session duration, tool usage
- **School-Specific Reports**: Admin-scoped analytics
- **Student Insights**: Personal dashboards, mood trends, goal tracking

### 9. Authentication & Security
- **NextAuth.js**: Session-based authentication with cookie management
- **Session Expiry**: Automatic session cleanup on expiration
- **Phone OTP Login**: Alternative authentication method
- **SSO Integration**: Single sign-on capability
